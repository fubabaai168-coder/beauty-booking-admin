# Beauty Admin v2 — Staff × Calendar UI Scope (Read-only)

目的：
在不新增資料模型、不接 API 的前提下，
提供「人員 × 行事曆」的視覺關聯，強化營運一眼可讀性。

---

## 一、原則（鎖定）
- 僅顯示，不可設定、不可編輯
- 不新增任何狀態、不新增任何操作
- 所有資料以 placeholder / mock 呈現
- 與既有 Calendar、Staff 視覺語言一致

---

## 二、Calendar 事件卡片擴充（Read-only）

### 顯示內容（新增）
- 指派人員（Staff）
  - 有指派：Avatar（圓形）＋姓名
  - 未指派：顯示「未指派」

### 顯示位置
- 事件卡片底部或右側（不影響既有資訊層級）
- Mobile 清單卡片中顯示為次要行

### 視覺規範
- Avatar：中性色背景，首字母
- 不因人員改變事件狀態色

---

## 三、人員詳情頁擴充（Read-only）

### 新增區塊：本週行事曆摘要
- 以清單顯示（日期 / 時段 / 服務 / 狀態 Badge）
- 點擊項目 → 導向該預約詳細頁

### 排序
- 依時間由近到遠

---

## 四、禁止事項（非常重要）
- 不設計「指派人員」操作
- 不做拖拉、不做即時更新
- 不新增 filter / search
- 不顯示衝突判斷

---

## 五、交付標準
- Calendar 事件卡片可看到人員資訊（或未指派）
- 人員詳情頁可看到本週行事曆摘要
- 所有頁面可 render、可點擊（只讀）
